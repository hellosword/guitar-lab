# Guitar Lab — 交互设计规格

> 日期：2026-05-01  
> 范围：MVP 阶段的指板、音名、唱名、颜色与联动规则。  
> 目的：让 Guitar Lab 中所有“位置 / 音名 / 唱名”的视觉表达保持一致。

---

## 1. 音名颜色系统

Guitar Lab 采用接近 Boomwhackers / Chroma-Notes 的音名颜色体系。

选择原因：

- 该体系常用于初学者音乐教育与彩色音符教学
- 颜色按音名字母稳定映射，符合“同名音跨八度同色”的记忆需求
- Soundslice 等音乐学习工具也支持 Boomwhackers colors

### 1.1 自然音颜色

| 音名 | 颜色名 | 用途 |
|------|--------|------|
| C | Red | Do / 主音候选 |
| D | Orange | Re |
| E | Yellow | Mi |
| F | Green | Fa |
| G | Teal | Sol |
| A | Indigo | La |
| B | Violet | Si |

> 注：不同资料对 G/A/B 的具体命名略有差异，如 G 可见 Teal / Dark Green / Blue，B 可见 Violet / Magenta / Pink。项目采用更适合深色 UI 的变体，但保持 C-D-E-F-G-A-B 的连续色相关系。

### 1.1.1 深色 UI 可读性规则

实际界面不直接使用高亮纯色或半透明色块，而使用适合深色背景的高对比色：

- 指板圆点内部文字统一使用白色
- E 等黄色系音名避免使用亮黄底 + 深色字，改用偏琥珀/棕黄的深色底
- F 与 G 必须明显区分：F 偏绿色，G 偏蓝绿色
- 半音使用相邻自然音的浅变体，但不能依赖透明度表达
- hover 淡化时仍需保留足够文字对比度

### 1.2 半音颜色

半音使用相邻自然音的“浅色 / 混合色”表达：

| 半音 | 视觉原则 |
|------|----------|
| C# | C 与 D 之间的偏浅暖色 |
| D# | D 与 E 之间的偏浅暖色 |
| F# | F 与 G 之间的偏浅青绿色 |
| G# | G 与 A 之间的偏浅蓝紫色 |
| A# | A 与 B 之间的偏浅紫色 |

MVP 阶段内部统一使用升号记法，不显示降号等音。

## 2. 指板标记规则

### 2.1 音名标记

- 每个位置显示其音名
- 自然音显示正色
- 半音显示浅色
- 默认只显示当前大调内音
- 用户可勾选“显示非当前大调内音”查看完整 12 音

### 2.2 唱名标记

- 每个位置显示当前大调下的首调唱名
- 调外音默认不显示
- 唱名颜色继承其对应音名颜色

示例：

```text
G 大调：G=Do, A=Re, B=Mi, C=Fa, D=Sol, E=La, F#=Si
```

## 3. 双向联动规则

### 3.1 指板 -> 映射面板

当用户点击指板位置时：

- 播放该位置音高
- 指板该位置高亮
- 右侧音名/唱名映射表同步高亮对应音名与唱名
- 当前点击区域显示位置、音名、唱名

### 3.2 映射面板 -> 指板

当用户悬停右侧映射面板中的某个音名或唱名时：

- 指板上所有对应音名的位置高亮
- 其它位置淡化
- 鼠标离开后恢复普通显示

该规则用于反向训练：

```text
看到 D / Sol -> 指板上所有 D 的位置
```

---

## 4. 不做事项

MVP 暂不做：

- 用户自定义颜色主题
- 降号体系显示
- 按和弦/音阶级数重新着色
- 色盲模式

后续如果颜色成为核心学习辅助，再补充可访问性配色与自定义主题。
