# 吉他指板记忆游戏 — 设计文档

> **项目目标**：为吉他爱好者构建一款轻量级指板记忆训练游戏，通过多种题型的反复练习，建立"位置-音名-唱名-音程-和弦"的直觉映射，最终提升即兴演奏与指板熟练度。
>
> **当前阶段**：乐理框架与交互设计定型期（技术栈尚未选定）。

---

## 1. 核心乐理模型

吉他指板记忆的本质，是以下六个集合之间的**双向映射**。

| 符号 | 含义 | 示例 |
|------|------|------|
| **P** | 位置 (Position) | 三弦2品、五弦0品 |
| **N** | 音名 (Note Name) | F#、Bb、E |
| **K** | 调式/调性 (Key) | G Major、A minor |
| **S** | 唱名 (Solfège) | Do、Re、Mi（依赖调式K）|
| **I** | 音程 (Interval) | 大三度、纯五度 |
| **C** | 和弦属性 (Chord) | Maj7、m7、属9 |
| **D** | 和弦内音级数 (Degree) | 根音、3音、5音、7音 |

> **乐理勘误备忘**：标准调弦下，F# 的正确位置包括一弦2品、四弦4品、六弦**2品**（六弦1品为F，非F#）。

---

## 2. 练习体系架构（三维参数体系）

游戏的组织方式以学习者的真实练琴逻辑为核心：**先选调性，再选玩法，最后限定范围**。

```
第一层：调性（Key）        ← 顶层入口
    └── 玩家首先确定"今天练什么调"：C / G / D / F / ...

    第二层：玩法类型（PlayMode）
        └── 进入调性后，选择练什么能力：
            ├── 指板音名记忆（位置 ↔ 音名）
            ├── 调式唱名记忆（位置/音名 ↔ 唱名）
            ├── 音程几何（两音之间的音程关系）
            ├── 和弦内音定位（和弦构成音在指板上的位置）
            ├── 音阶把位自由（任意把位、任意调的音阶图形）
            └── 多声部分析（指弹声部功能识别）

        第三层：范围（Range）
            └── 最后限定指板区域：
                ├── 低把位（开放把位 / 0-3品 / 0-4品 / 0-5品）
                ├── 中把位（3-7品 / 5-8品）
                ├── 高把位（8-12品）
                └── 全指板
```

### 2.1 为什么以调性为第一入口？

绝大多数流行歌曲首先确定的是调号。学习者在建立指板直觉时，**在同一调性内逐步叠加能力**（先会G大调的音名，再会G大调的唱名，再会G大调的音程...），比**在同一能力下跨调乱序跳跃**更符合认知规律。

### 2.2 三维参数的关系

三个维度是正交的，任意组合都是合法的练习配置：

| 维度 | 决定什么 | 示例 |
|------|---------|------|
| **Key** | 当前可用的音名池 | G大调下，有效音为 G A B C D E F# |
| **PlayMode** | 训练的认知能力 | P→N（位置→音名）、N→P（音名→位置） |
| **Range** | 物理指板边界 | 0-4品、仅低音弦组（5-6弦） |

**核心生成规则**：`有效位置池 = Range 内所有物理位置 ∩ Key 调性音阶内的位置`

> **示例**：G大调 × 0-4品<br>
> 物理位置共30个（5品×6弦），但经G大调音阶过滤后，有效位置约18个（只保留音名为 G/A/B/C/D/E/F# 的位置）。F(1品) 和 G#(4品) 等不在G大调音阶内的位置被过滤掉。

### 2.3 与原有"6层认知难度"的关系

原有的 Level 1~6 并未废弃，而是**从"线性关卡"转换为"玩法类型清单"**。每种 PlayMode 都可以在每个调性、每个范围下独立练习：

| 原层级 | 新 PlayMode 名称 | 核心映射 |
|--------|-----------------|---------|
| Level 1 | **指板音名记忆** | P ↔ N |
| Level 2 | **调式唱名记忆** | P+K ↔ S |
| Level 3 | **音程几何** | P₁ + I ↔ P₂ |
| Level 4 | **和弦内音定位** | C + D + RootP ↔ P |
| Level 5 | **音阶把位自由** | K + Scale + Position ↔ AllP |
| Level 6 | **多声部分析** | Polyphonic |

玩家不是在"通关 Level 1"，而是在"掌握 C大调的指板音名记忆、G大调的指板音名记忆..."。当进入一个新的调性（如D大调）时，即使已在C大调掌握了音程与和弦，仍建议从该调的**音名记忆**重新开始建立映射。

### 2.4 预设（Preset）机制

为避免玩家每次都要手动配置三维参数，系统提供**预设卡片**作为快捷入口：

| 预设名称 | 参数组合 |
|---------|---------|
| 开放把位C大调音名识别 | C大调 × P→N × 0-3品 |
| G大调基础音名 | G大调 × P→N × 0-4品 |
| 开放把位C大调音名定位 | C大调 × N→P × 0-3品 |
| 第5把位C大调音阶 | C大调 × ScaleMap × 第5把位 |

预设只是参数组合的快捷方式，玩家可以基于预设随时微调参数并保存为自定义配置。

## 3. 玩法类型（PlayMode）详细定义

以下是将原有"6层认知难度"转换后的 PlayMode 清单。每个 PlayMode 的输入输出规则不变，但可在任意调性、任意范围下通过配置生成具体练习。

### PlayMode 1：指板音名记忆（P ↔ N）
建立指板与绝对音高的直觉，是整个游戏的底座。

| 编号 | 题目给出 | 用户回答 | 示例 |
|------|----------|----------|------|
| 1.1 | **P** | **N** | 三弦2品 → A |
| 1.2 | **N** | **P**（所有位置） | F# → 一弦2品、四弦4品、六弦2品… |

> **排除项**：不做 P + N → 判断对错（1.3），该题型偏离核心映射训练目标。
> **N→P 变体**：支持全范围多点点击，也支持弦组限定（如"仅在低音弦组内找出所有C"）。

### PlayMode 2：调式唱名记忆（P+K ↔ S）
引入相对音高概念，连接固定调与首调思维。

| 编号 | 题目给出 | 用户回答 | 示例 |
|------|----------|----------|------|
| 2.1 | **P + K** | **S** | G大调下，三弦2品 → Re |
| 2.2 | **S + K** | **P**（所有位置） | G大调的Re → 所有A的位置 |
| 2.3 | **N + K** | **S** | G大调下，F# → Si |

> **排除项**：不做 P + S + K → 判断对错（2.4）。

### PlayMode 3：音程几何（P₁ + I ↔ P₂）
训练指板上的音程几何感，脱离死记硬背。

| 编号 | 题目给出 | 用户回答 | 示例 |
|------|----------|----------|------|
| 3.1 | **P₁ + I** | **P₂** | A（三弦2品）上方大三度 → 三弦6品(C#) 或 二弦2品(C#) |
| 3.2 | **P₁ + P₂** | **I** | 三弦2品 → 二弦2品 是什么音程？→ 大三度 |
| 3.3 | **N₁ + I** | **N₂** | E上方纯五度 → B |
| 3.4 | **P₁ + I + 指定弦** | **P₂** | 限制在单弦上找音程 |

### PlayMode 4：和弦内音定位（C + D + RootP ↔ P）
从单音进入和声思维。

| 编号 | 题目给出 | 用户回答 | 示例 |
|------|----------|----------|------|
| 4.1 | **C + D + RootP** | **P** | 以五弦3品(C)为根音，大三和弦的5音 → 四弦5品(G) |
| 4.2 | **P + C + RootP** | **D** | 某位置在Cmaj7里是几音？ |
| 4.3 | **RootP + C** | **All P** | 给定根音位置，构建完整和弦指法 |
| 4.4 | **C + D** | **All P** | Cmaj7的3音(E)在全指板的所有位置 |

### PlayMode 5：音阶把位自由（K + Scale + Position ↔ AllP）
整合调式、音阶形状与指板视觉。

| 编号 | 题目给出 | 用户回答 | 示例 |
|------|---------|---------|------|
| 5.1 | **K + ScaleType + P** | In/Out? | G大调里，二弦1品(C)是不是音阶音？→ 否 |
| 5.2 | **K + ScaleType + 弦** | **All P** | G大调在五弦上的所有音阶音 |
| 5.3 | **K + ScaleType + 把位** | **All P** | G大调第5把位的音阶图形 |
| 5.4 | **S序列 + K** | **P序列** | 按顺序点击 do→re→mi→fa→sol 的路径 |
| 5.5 | **ChordProgression + K** | **把位切换** | ii-V-I 进行在低把位到高把位间平滑切换 |

### PlayMode 6：多声部指弹分析（高阶 / 后期解锁）
从单音/音程认知进入**多声部功能认知**。

| 编号 | 题目给出 | 用户回答 | 示例 |
|------|----------|----------|------|
| 6.1 | **P_tab_polyphonic + Chord** | **V**（声部功能） | 标记每个音是 Melody / Bass / Accompaniment |
| 6.2 | **Chord + Melody_line** | **Bass_line** | C-Am-F-G 进行中，给定旋律，补全低音根音线 |
| 6.3 | **P_tab_polyphonic** | **Chord + Bass_Note** | 从指弹谱识别当前和弦及低音 |
| 6.4 | **Chord + Bass_line** | **P_tab**（补全编配） | 给定和弦和低音，在指板上填充伴奏内声部 |

> **前置依赖**：PlayMode 1~5 及锚点 ①③④。建议作为后期解锁或独立扩展包。

## 4. 位置(P)的三种媒介形式

| 形式 | 符号 | 用途特点 |
|------|------|---------|
| **文本位置** | P_text | "3弦2品"。信息密度低，适合辅助说明，不适合复杂图形 |
| **六线谱** | P_tab | 适合展示旋律走向、单音位置、和弦按法图。吉他手最熟悉 |
| **指板图** | P_board | 可高亮单点/多点，**唯一支持"点击输入"的媒介** |

### 媒介分工规则
- **文本与六线谱**：通常仅作为**题目展示**（输出媒介），不用于玩家输入。
- **指板图**：既可作为**题目展示位置**（高亮某点让玩家猜），也可作为**输入器**（让玩家点击指板位置来答题）。

## 5. 答题交互规范

答题交互必须由**答案类型**决定，不可为了形式多样性牺牲体验。

| 用户要回答什么 | 最佳交互 | 次选交互 | 避免 |
|--------------|---------|---------|------|
| **音名** (C / F# / Bb) | **字母+升降号分区域选择器** | 文字输入(2字内) | 12选1卡片 |
| **唱名** (Do/Re/Mi…) | **7个固定首调唱名按钮** | 文字输入 | 升降号选项 |
| **位置 - 单位置** | 指板图点击 | — | 文字输入 |
| **位置 - 多点** | 指板图**多点点击** | — | 文字输入（痛苦）|
| **音程** (大三度/纯五度…) | 选项按钮 / 滚轮 | 文字输入 | — |
| **和弦内音级数** (根/3/5/7音) | 选项按钮 | 文字输入 | — |
| **判断对错** | 是/否 或 红绿双按钮 | — | — |

> **UX铁律**：只要答案是"位置"（尤其是多个位置），**指板图必须是唯一的答题界面**。

### 唱名选择器详细设计（首调唱名法 / Movable-do）

游戏采用**首调唱名法**，唱名永远只有7个纯净选项，不附加升降号：

```
   Do   Re   Mi   Fa   Sol   La   Si
```

**核心规则**：
- 唱名随调式主音移动（G大调中 G=Do, A=Re, F#=Si）
- 调式内的升降号由**调号统一处理**，不体现在唱名上
  - ✅ G大调的 F# → 唱 **Si**，不是 #Fa
  - ✅ D大调的 C# → 唱 **Si**，不是 #Do
- 升降号仅出现在**音名输入**中，不出现在**唱名输入**中

**为什么不用固定调+升降号？**
- 吉他初学者普遍使用首调思维（"G大调的Si在哪"比"G大调唱#Fa"更自然）
- 首调法下自然大/小调、五声音阶均只需7个按钮，认知负荷最低
- 固定调的升降唱名（#Do, bRe 等）属于高阶视唱练耳范畴，如需覆盖可在后期扩展

**适用题型**
- P + K → S（位置+调式→唱名）
- S + K → P（唱名+调式→位置）
- N + K → S（音名+调式→唱名）

---

### 音名选择器详细设计（推荐方案）

音名输入采用**两区选择器**，而非12选1卡片，原因：
- 符合乐理直觉（人脑处理音名是先辨"字母"再辨"升降"）
- 降低认知负荷（7选1 + 3选1，优于12选1）
- 具有专业感（类似DAW音高选择器）

**第一区域：字母音名（7选1）**
```
   C    D    E    F    G    A    B
```

**第二区域：升降号（3选1）**
```
   ♮(自然)    #(升)    b(降)
```
- 默认选中 `♮`（自然）。
- **禁用规则**：
  - 选中 **E / B** 时，`#` 按钮置灰禁用（E#/B# 在常规乐理中极少使用）
  - 选中 **F / C** 时，`b` 按钮置灰禁用（Fb/Cb 极少使用）
  - 这样用户能自然学到"E一般不升、F一般不降"的乐理常识

**提交方式**
- **快节奏模式**：选完两个区域后**自动提交**（无需确认按钮）
- **防误触模式**：提供确认按钮，或支持回车/空格快捷键提交
- **修改支持**：已提交后可回溯修改，升降号状态在切换字母时保留

**等音处理**
- 游戏中**固定使用升号记法**（如 F#），不接受等音替代（如 Gb）
- 若用户选了 G + b，系统提示"通常记为 F#"或直接禁用该组合

**适用题型**
- P → N、P_tab → N（六线谱视奏）、H → N（听力）等所有需要回答音名的场景

### 核心玩法 × 交互映射

| 玩法 | 题目展示 | 用户输入 | 体验评级 |
|------|---------|---------|---------|
| **P → N** | P_board / P_tab / P_text | N_text / N_option | ⭐ 完美。三种题目形式可轮换防疲劳 |
| **N → P** | N_text | P_board 多点点击 | ⭐ 完美。指板图作为输入器的最佳场景 |
| **P + K → S** | P_board + K_text | S_option 7选1 | ⭐ 极好。零输入成本 |
| **S + K → P** | S_option + K_text | P_board 多点点击 | ⭐ 极好 |
| **P₁ + I → P₂** | P_board + I_option | P_board 点击目标 | ⭐ 极好。音程几何可视化 |
| **P₁ + P₂ → I** | P_board 高亮两点 | I_option | ⭐ 极好 |
| **Chord + Degree + RootP → P** | C_text + D_option + P_board | P_board 点击 | ⭐ 极好 |
| **RootP + ChordType → Pattern** | P_board + C_text | P_board 多点点击（点亮所有构成音）| ⭐⭐ **最佳多点击玩法** |
| **K + Scale + Position → AllP** | K_text + Scale_option | P_board 多点点击（绘制音阶地图）| ⭐⭐ **把位记忆王牌玩法** |
| **S序列 + K → P序列** | S序列 + K_text | P_board 按顺序点击 | ⭐⭐ **旋律路径训练** |

## 6. 核心能力锚点（重点标注）

以下四种映射训练被定义为**锚点能力**，它们分别对应吉他演奏中最高频的四个真实场景，应在游戏设计中给予最高优先级和足够的训练量。

### 锚点 ①：六线谱视奏 → 音名/唱名（Sight Reading）
**对应能力**：拿起谱子就能知道自己在弹什么音，而不是只记手指动作。
- **Level 1 映射**：P_tab → N（六线谱上的数字 → 音名）
- **Level 2 映射**：P_tab + K → S（六线谱 + 调式 → 唱名）
- **为什么关键**：六线谱是吉他世界的通用语，但它本身不携带音高信息。很多吉他手能弹复杂曲子却说不出自己在弹什么音——这就是视奏能力的断层。
- **训练形式**：六线谱上随机出现数字，限时回答音名/唱名；或滚动播放六线谱，持续跟答。

### 锚点 ②：音程几何直觉（Interval Geometry）
**对应能力**：即兴时不必思考音名，凭"形状"就能跳到正确的音。
- **Level 3 映射**：P₁ + I → P₂（已知位置 + 音程 → 目标位置）
- **为什么关键**：指板不是逐格记忆的，而是靠"几何形状"记忆的。知道"大三度在跨弦时的视觉形状"比知道"A上方大三度是C#"更直接。
- **训练形式**：指板图上高亮一个根音，要求点击所有可能的"大三度"位置（跨弦+同弦）；或给定两个高亮点，判断音程。

### 锚点 ③：和弦内音定位（Chord Tone Targeting）
**对应能力**：在和弦进行中，solo或编配时准确落在和弦音上，声音瞬间"稳"了。
- **Level 4 映射**：C + D + RootP → P（和弦 + 音级 + 根音位置 → 目标位置）
- **为什么关键**：爵士/指弹/即兴伴奏的骨架。知道Cmaj7的3音和7音在哪里，比会弹整个Cmaj7和弦指型更有音乐性。
- **训练形式**："根音在此，找出C7的b7音"；或"点亮Cmaj7的所有构成音"。

### 锚点 ④：把位音阶自由（Position Freedom / CAGED）
**对应能力**：不再被"第一把位Mi型音阶"困住，任意把位、任意调都能即兴。
- **Level 5 映射**：K + ScaleType + Position → AllP（调式 + 音阶类型 + 把位 → 完整图形）
- **为什么关键**：这是吉他手从"弹曲子"到"玩音乐"的分水岭。CAGED系统的本质就是"把位音阶自由"。
- **训练形式**："G大调，第7把位，点亮所有音阶音"；或限制在3品宽的"把位框"内找音。

### 锚点 ⑤：指弹声部分离与功能分析（Fingerstyle Voice Separation）
**对应能力**：看到指弹六线谱，瞬间分解出"谁在唱旋律、谁在走低音、谁在弹伴奏"，从而能够推导、改编甚至即兴编配。
- **Level 6 映射**：P_tab_polyphonic + Chord → V_map（多声部六线谱 + 和弦名 → 每个音的声部功能标记）
- **为什么关键**：指弹的本质是"一个人弹一个乐队"。机械记忆每根手指按哪里是低效的；真正学懂的人理解的是功能——这个音是旋律(Melody)、是低音(Bass)、还是和声填充(Accompaniment)。
- **训练形式**：
  - 给定一小节指弹六线谱+当前和弦，标记每个音属于 M/B/A
  - 给定和弦进行+旋律线，在指板上补全合理的低音线
  - 给定指弹谱（无和弦标记），识别和弦与低音进行
- **前置依赖**：必须已掌握 Level 1~5 及锚点 ①③，建议作为后期/高阶解锁内容。

## 7. 范围（Range）与调性过滤

### 7.1 范围的维度

范围是对物理指板的限制，作为三维参数中的第三层：

| 维度 | 示例 |
|------|------|
| **品格范围** | 0-3品、0-4品、0-5品、5-8品、全指板 |
| **弦组范围** | 全6弦、低音弦组（5-6弦）、中音弦组（3-4弦）、高音弦组（1-2弦） |
| **把位范围** | 开放把位、CAGED-C把位、CAGED-A把位 |

### 7.2 调性过滤：有效位置池

生成器的核心逻辑：

```
步骤1：确定 Key → 得到该调音阶音名集合（如 G大调 = {G,A,B,C,D,E,F#}）
步骤2：确定 Range → 得到范围内所有物理位置
步骤3：过滤 → 只保留音名在音阶集合内的位置
步骤4：从过滤后的有效位置池里随机出题
```

不同调性在相同 Range 下的有效位置数量不同：

| 调性 | 0-3品有效位置 | 0-4品有效位置 |
|------|--------------|--------------|
| C大调 | ~18个（无升降号） | ~24个 |
| G大调 | ~14个（缺部分F#） | ~18个 |
| F大调 | ~16个（含Bb，缺部分自然音） | ~20个 |

> **示例**：G大调 × 0-4品<br>
> 6弦：0(E)✓、1(F)✗、2(F#)✓、3(G)✓、4(G#)✗<br>
> 5弦：0(A)✓、1(A#)✗、2(B)✓、3(C)✓、4(C#)✗<br>
> 4弦：0(D)✓、1(D#)✗、2(E)✓、3(F)✗、4(F#)✓<br>
> 以此类推，最终得到有效位置约18个。

### 7.3 调性解锁顺序

按调号数量从少到多排列，作为推荐的学习顺序：

```
C（0个） → G（1个#） → D（2个#） → A（3个#）
        → F（1个b） → Bb（2个b） → Eb（3个b）
```

玩家可以在任意已解锁调性下自由选择 PlayMode 和 Range。

### 7.4 认知负荷与推荐路径

系统的推荐路径按**认知负荷**排序，将三维空间"拍平"为一维渐进路线：

```
认知负荷 ≈ PlayMode复杂度 + Range大小 + 调号数量
```

| 推荐顺序 | 配置 | 负荷说明 |
|---------|------|---------|
| 1 | C大调 × P→N × 0-3品 | 无升降号、小范围、单向映射 |
| 2 | G大调 × P→N × 0-4品 | 引入1个升号（F#） |
| 3 | C大调 × N→P × 0-3品 | 切换为空间记忆（多点点击） |
| 4 | C大调 × P+K→S × 0-3品 | 引入唱名概念 |
| 5 | D大调 × P→N × 0-5品 | 2个升号 + 范围扩展 |
| 6 | C大调 × Interval × 0-5品 | 引入音程关系 |

> **设计原则**：范围扩大与调号增加必须**逐步解锁**，避免一次性开放全指板或多升降号调性，否则会产生认知过载，破坏心流。

## 9. 反人类组合排除清单

以下组合理论上存在，但应从设计中**主动剔除**：

| 组合 | 排除原因 |
|------|---------|
| N → P_text | 让玩家打字输入多个位置"一弦2品、四弦4品、六弦2品…" |
| P_multi → N | 指板上高亮多个点问"这是什么音名"（答案不唯一，混乱）|
| S + K → P_tab | 六线谱无法作为"点击输入位置"的媒介 |
| ChordPattern → P_tab编辑 | 让玩家画六线谱和弦图，偏离训练目标 |
| 任意多点位置题 → 文字输入 | 只要答案涉及多点，必须用指板图点击 |

## 9. 设计原则总结

1. **调性第一入口**：玩家首先选择调性，在同一调性内逐步叠加能力，而非跨调乱序练习。
2. **正交三维参数**：Key（调性）、PlayMode（玩法类型）、Range（指板范围）三个维度独立配置，任意组合均合法。
3. **调性过滤位置池**：生成器以"调性音阶"过滤范围内的物理位置，只出有效音。
4. **指板图是核心交互器**：所有涉及位置的输入，必须通过指板图点击完成。
5. **答案类型决定输入方式**：音名/唱名用选项或短文本；位置用指板点击；音程/级数用固定选项。
6. **正反向都练**：不仅考 P→N，也必须考 N→P；不仅考 P+K→S，也必须考 S+K→P。
7. **预设作为快捷入口**：常用参数组合打包为预设卡片，降低配置成本。
8. **听力二期解锁**：保持MVP轻量，听力训练作为后续扩展。
