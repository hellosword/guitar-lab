# 移动端交互 Review 报告

> 日期：2026-05-04  
> 页面版本：v0.0.36  
> 范围：手机竖屏下的练习页、指板输入、底部导航、设置 Sheet  
> 结论：当前移动端不是“很烂”，已经有可用的 Focus Mode 雏形；但仍偏“桌面页面移动化”，距离“日常练习器”还差一轮交互打磨。

---

## 1. 总体判断

当前移动端已经完成了几个关键正确方向：

- 顶部有轻量上下文栏，能看到当前练习和调性。
- 桌面端的大型练习模式区已经收进设置 Sheet。
- 底部有“练习 / 速查 / 弱点”三类任务导航。
- 首屏可以看到题目、指板和作答按钮，用户不是一打开就被设置淹没。

主要问题不在“有没有移动端”，而在“练习节奏是否顺手”。现在的核心体验仍然像桌面卡片纵向堆叠，移动端只是把复杂度收纳了一部分，还没有完全围绕单手、短时、高频练习优化。

## 3. 主要问题

### 3.1 指板在手机竖屏被无提示裁切

严重程度：高

现象：

- 390px 宽手机视口下，指板 SVG 保持较大的可点宽度。
- 页面级没有横向溢出，但指板卡片内部实际只露出左侧部分。
- 0-4 品练习中，右侧品位可能不在首屏可见范围内。

影响：

- 对“音名 -> 位置”“唱名 -> 位置”题型尤其危险。
- 用户可能误以为目标位置不存在，或者没有意识到需要横向滑动。
- 这会直接影响答题正确性，而不只是视觉瑕疵。

建议：

- 短期保留横向滚动，但必须提供明确提示、边缘渐隐和可访问文案。
- 中期设计移动端指板专属输入模式，例如横屏练习、放大输入、分段品位导航。
- 验收时必须确认页面级 `scrollWidth` 不超过 viewport，同时指板内部可横向滚动且可发现。

当前状态：

- 已开始修复：指板组件加入内部横滑提示与回归测试。
- 后续仍需评估渐隐状态是否会在滚到最右后误导用户。

### 3.2 底部导航与练习动作抢位置

严重程度：高

现象：

- 移动端底部固定导航常驻。
- “播放音高”“下一题”“反馈状态”等练习动作仍在内容流中。
- 在首屏底部，练习动作容易贴近或被底部导航压住。

影响：

- 练习动作比跨页面导航更高频，优先级却没有被固定到底部。
- 用户刚答完题时，最需要的是听音、看反馈、进入下一题，而不是切到速查或弱点。

建议：

- 将“练习动作条”和“主导航”分层。
- 练习中优先固定：播放音高、反馈、下一题。
- 主导航可以降级为更薄的 tab、可隐藏导航，或只在非答题状态保持显著。

### 3.3 首屏仍然偏桌面卡片纵向堆叠

严重程度：中高

现象：

- 题目卡、指板卡、Answer 卡、练习详情卡依次堆叠。
- 卡片边框、英文小标题和重复上下文较多。
- 首屏虽然能练，但视觉重心分散。

影响：

- 用户需要在多个卡片之间理解“现在看哪里、点哪里”。
- 对高频练习来说，卡片结构比任务流更重。

建议：

- 移动端练习页改为连续任务面板：
  - 顶部：当前通路、调性、设置入口
  - 中部：题号、题干、目标
  - 主区：指板 / 六线谱 / 音名大字卡
  - 下方：作答按钮
  - 底部：练习动作条
- 压缩或移除移动端的 `ANSWER` 英文标签和重复上下文胶囊。

### 3.4 设置 Sheet 信息密度偏高

严重程度：中

现象：

- 设置 Sheet 同时承载练习模式、图选通路、当前页面、调性、唱名显示、音色、版本、构建标识。
- 这比原来的桌面工具栏更整洁，但仍像“控制台收进抽屉”。

影响：

- 首次用户打开设置后，可能无法判断哪些是日常高频项。
- 低频信息和高频切换项混在一起，增加移动端操作负担。

建议：

- 常用项优先：练习组、方向、调性。
- 次常用项折叠：唱名显示、音色。
- 低频项后置：图选通路、版本、构建标识。
- 模式切换若会重开当前练习，应有清晰反馈或确认语义，例如“开始此练习”。

### 3.5 移动端仍和桌面 Shell 耦合较深

严重程度：中

现象：

- 移动端壳层目前仍在 `src/desktop/DesktopApp.tsx` 中组合。
- 组件内部通过 `isDesktopLayout` 分支处理移动端差异。

影响：

- 短期可接受，但后续移动端任务流继续演化时，桌面布局和移动布局会互相牵制。
- 交互差异越大，越不适合继续堆条件 class。

建议：

- 保持 domain、题库、校验、记忆系统共享。
- 逐步抽出移动端 Shell，例如 `src/mobile/MobilePracticeShell.tsx`。
- 不要复制业务逻辑，只拆 UI 组合和任务流。

## 4. 大改方向

建议把移动端定义为“练习器”，而不是“窄屏工作台”。

推荐信息架构：

```text
顶部上下文
  当前练习 / 当前调性 / 设置

题目区
  第 n / 20 题
  一句话任务
  当前目标

主输入区
  指板 / 六线谱 / 音名 / 唱名

作答区
  单选按钮 / 指板点击

底部练习动作
  播放音高 / 反馈状态 / 下一题

次级导航
  练习 / 速查 / 弱点
```

关键原则：

- 练习动作优先于跨页面导航。
- 指板输入优先于装饰卡片。
- 设置后置但不能隐藏到难找。
- 反馈应贴近刚发生的答题动作。

## 5. 建议修复顺序

### P0：指板输入可发现性

目标：

- 手机竖屏下指板可横向滚动或以更合适方式完整访问。
- 用户能明确知道右侧还有品位。
- 页面整体不出现横向溢出。

验收：

- 390px 视口下 `document.documentElement.scrollWidth <= clientWidth + 1`。
- 指板内部滚动容器 `scrollWidth > clientWidth` 时必须有可见提示。
- 0-4 品范围内所有目标品位可访问。

### P1：底部练习动作条

目标：

- 播放音高、下一题、反馈状态不再被底部导航挤压。
- 答题后的下一步动作更明显。

验收：

- 错题后不用滚动即可看到“下一题”。
- 正确自动进入下一题时有短暂反馈但不挡住主输入。
- 底部导航不遮挡任何练习按钮。

### P2：首屏信息层级瘦身

目标：

- 减少移动端卡片边框和重复标签。
- 让题目、输入、作答成为第一视觉层级。

验收：

- 390px 首屏内能完整看到题干、主要输入区和至少一部分作答区。
- 移动端不再展示不必要的桌面式说明文案。

### P3：设置 Sheet 渐进化

目标：

- 常用设置优先，低频配置折叠或后置。
- 切换练习模式的结果更清楚。

验收：

- 首屏 Sheet 只出现最常用设置。
- 版本/构建信息不抢占练习设置空间。
- 切换会重开练习时，用户能理解发生了什么。

## 7. 结论

当前移动端已经从“PC 页面缩小版”走到了“Focus Mode 第一版”，这是正确路线。下一轮不需要推倒重来，但要坚定地把移动端从桌面布局里继续剥离出来：先保住指板输入，再解决底部练习动作，然后瘦身首屏和设置 Sheet。这样它才会从“能打开练”变成“愿意每天练”。
