# 练习通路式交互框架规格

> 日期：2026-05-03  
> 当前状态：第一版已实现于 v0.0.18，常驻布局校准于 v0.0.19，图选弹窗校准于 v0.0.20，模式/动作层级校准于 v0.0.21，模式说明去重于 v0.0.22，图选入口收纳于 v0.0.23，通路视图 tab 化于 v0.0.24，日常练习密度优化于 v0.0.25，训练组/方向层级改造待发布  
> 范围：Guitar Lab MVP 的主导航、练习选择、弱点入口与速查定位。  
> 目标：把“练习类型”从按钮列表升级为更符合认知模型的有向图选择器。

---

## 1. 背景

当前界面的核心区由三个维度共同决定：

- 主行为：练习 / 指板速查 / 弱点地图
- 全局上下文：当前大调
- 练习类型：综合、指板音名、指板唱名、六线谱音名、六线谱唱名、音名唱名、音名定位

这种结构在 MVP 早期足够直接，但随着题型、弱点数据和速查工具增加，它容易变成一组平铺按钮。用户需要自己判断每个按钮是在切换页面、切换练习模式，还是查看某类数据。

本规格采用新的产品模型：

```text
全局上下文：调性 + 唱名显示偏好

主行为：
练习：在认知通路上做反应训练
速查：查看稳定认知地图和辅助表

练习内部：
每条认知通路 = 开始练习 + 查看对应弱点
```

## 2. 核心概念

### 2.1 认知节点

MVP 阶段使用四类认知节点：

| 节点 | 含义 | 备注 |
|------|------|------|
| 指板位置 | 弦 + 品形成的位置感 | 可作为输入，也可作为目标 |
| 六线谱 | 谱面位置表示 | 当前只作为输入，不作为目标 |
| 音名 | C/D/E/F/G/A/B 与升号音 | 连接位置和唱名的核心中间层 |
| 唱名 | 当前调性下的首调唱名 | 显示可为 Do/Re/Mi 或 1/2/3 |

### 2.2 认知通路

练习不是某个高级分类下的菜单项，而是节点之间的有向边。

当前已实现或正在打磨的通路：

| 通路 | 对应现有模式 | 说明 |
|------|--------------|------|
| 指板位置 -> 音名 | `board-to-note` | 看到指板位置，反应音名 |
| 指板位置 -> 唱名 | `board-to-solfeggio` | 看到指板位置，反应当前调唱名 |
| 六线谱 -> 音名 | `tab-to-note` | 看到六线谱，反应音名 |
| 六线谱 -> 唱名 | `tab-to-solfeggio` | 看到六线谱，反应当前调唱名 |
| 音名 -> 唱名 | `note-to-solfeggio` | 看到音名，反应当前调唱名 |
| 唱名 -> 音名 | `solfeggio-to-note` | 看到当前调唱名，反应对应音名 |
| 音名 -> 指板位置 | `note-to-positions` | 看到音名，在指板上找所有位置 |
| 唱名 -> 指板位置 | `solfeggio-to-positions` | 看到当前调唱名，在指板上找所有对应位置 |

`mixed` 综合练习不是图中的节点，也不是一条边。它是“混合当前已启用通路”的特殊入口。

### 2.3 训练组与方向

日常练习页不再把所有通路平铺成一排按钮，而是分两层：

| 训练组 | 方向 |
|--------|------|
| 综合 | 全系统混合 |
| 指板记忆 | 音名组：位置 -> 音名、音名 -> 位置、混合；唱名组：当前调选择、位置 -> 唱名、唱名 -> 位置、混合 |
| 音名唱名 | 音名 -> 唱名、唱名 -> 音名、混合 |
| 六线谱 | 六线谱 -> 音名、六线谱 -> 唱名、混合 |

第一层回答“练哪组关系”，第二层回答“练哪个方向”。这样新增通路时不会把日常界面推回 8 个以上平铺按钮。

## 3. 信息架构

### 3.1 顶层结构

顶层主导航只保留两个主行为：

```text
[练习] [速查]
```

全局上下文保持稳定可见：

```text
调性：[G 大调] [C 大调]
唱名显示：[Do Re Mi] [1 2 3]
版本号：v0.0.x
```

`Do Re Mi / 1 2 3` 是显示偏好，不参与主导航，也不改变内部唱名枚举。

### 3.2 练习页

练习页包含三层：

```text
练习通路图
当前通路操作区
实际练习区 / 对应弱点区
```

练习通路图负责选择“练什么”。实际题面、答题区、反馈区仍沿用现有练习闭环。

### 3.3 速查页

速查页收纳全局认知工具。当前第一版包含：

- 指板速查
- 当前调音名 / 唱名映射

未来可以扩展：

- 五度圈
- 调内音阶表
- 音程关系图
- 和弦构成表

指板速查的产品地位是全局速查工具，而不是某个练习模块下的页面。练习页中的“指板记忆”则表示围绕指板位置的反应训练组。

## 4. 练习通路图

### 4.1 固定图布局

第一版使用固定布局，不使用力导向图，不做复杂动画。

建议结构：

```text
          [六线谱]
          ↘     ↘
           ↘     ↘
[指板位置] -> [音名] -> [唱名]
      ↘                  ↑
       ↘-----------------┘
```

实际视觉可以根据空间调整，但必须保留：

- 节点清晰可读
- 方向箭头明确
- 当前选中通路高亮
- 支持鼠标和触控点击

### 4.2 节点视觉

节点建议使用名称 + 轻量图标：

| 节点 | 建议表达 |
|------|----------|
| 指板位置 | 小型指板图标或弦品网格 |
| 六线谱 | 六条横线图标 |
| 音名 | `C D E` |
| 唱名 | 跟随全局显示：`Do Re Mi` 或 `1 2 3` |

### 4.3 边视觉

边是主要可交互对象。

| 状态 | 表现 |
|------|------|
| 默认 | 灰色细线 + 箭头 |
| hover / focus | 边和两端节点轻微提亮 |
| 当前选中 | 使用主强调色，高亮方向 |
| 有弱点数据 | 边附近显示轻量热度标记 |
| 暂不可用 | 降低透明度，不响应点击 |

第一版不需要在边上直接塞长文本，避免图形拥挤。通路说明放在下方操作区。

## 5. 当前通路操作区

点击一条通路后，显示该通路的说明和动作。

示例：

```text
当前选择：音名 -> 唱名
在当前调里，看到音名后快速反应唱名。

[开始练习] [查看弱点]
```

行为规则：

- 点击“开始练习”：进入该通路对应的练习模式，并重开本轮练习。
- 点击“查看弱点”：打开该通路对应的弱点视图。
- 如果该通路暂时没有弱点实现，“查看弱点”显示为不可用或“暂无数据”。
- 切换调性时保留当前选中的通路，但练习题需要按当前规则重开。
- 切换唱名显示偏好时不重开练习，只更新显示文本。

综合练习作为特殊入口显示：

```text
综合练习：混合当前已启用通路
[开始综合练习]
```

综合练习第一版可以不提供独立弱点入口。

## 6. 弱点入口定位

弱点不再作为顶层主页面，而是挂在对应练习通路下。

```text
指板位置 -> 音名
├── 开始练习
└── 查看弱点

音名 -> 指板位置
├── 开始练习
└── 查看弱点

音名 -> 唱名
├── 开始练习
└── 查看弱点
```

当前已有指板弱点地图可以先服务两条相关通路：

- 指板位置 -> 音名
- 音名 -> 指板位置

未来新增其它弱点统计时，也应挂在对应通路下，而不是扩展一个越来越重的全局“数据中心”。

## 7. 状态模型建议

建议界面层状态拆成：

```ts
type MainView = 'practice' | 'reference';

type PracticePathId =
  | 'mixed'
  | 'position-note-mixed'
  | 'position-solfeggio-mixed'
  | 'note-solfeggio-mixed'
  | 'tab-reading-mixed'
  | 'position-to-note'
  | 'position-to-solfeggio'
  | 'tab-to-note'
  | 'tab-to-solfeggio'
  | 'note-to-solfeggio'
  | 'solfeggio-to-note'
  | 'note-to-position'
  | 'solfeggio-to-position';

type PracticeSubView = 'train' | 'weakness';
```

关键状态：

```ts
currentView: MainView;
currentKey: 'C' | 'G';
solfeggioDisplayMode: 'syllable' | 'number';
selectedPracticePath: PracticePathId;
practiceSubView: PracticeSubView;
```

状态语义：

- `currentView` 决定用户是在练习还是速查。
- `currentKey` 是全局音乐上下文，影响练习、速查和弱点解释。
- `solfeggioDisplayMode` 是全局显示偏好。
- `selectedPracticePath` 决定当前练习通路。
- `practiceSubView` 决定当前通路下是训练还是弱点。

## 8. 第一版落地范围

第一版只调整交互框架，不新增学习算法。

必须完成：

- 顶层主导航调整为“练习 / 速查”。
- 将指板速查迁移到“速查”视图。
- 练习页增加固定布局的通路图选择器。
- 当前练习模式按钮替换为通路图 + 当前通路操作区。
- 弱点地图入口移动到对应通路下。
- 保留现有所有题型能力。
- 保留现有右侧详情区、播放音频、答题反馈和统计。
- 保留现有调性规则与唱名显示偏好。

v0.0.19 校准：

- 日常练习页不常驻展示完整通路图，避免挤占题面和答题区。
- 常驻区域改为紧凑水平题型按钮，并保留当前通路说明。
- 通过“图选”按钮打开通路图弹窗，在需要切换或理解关系时再查看 graph。
- 图中点击通路后直接切换练习模式并关闭弹窗。

v0.0.20 校准：

- 图选弹窗移除右侧 `Current Path` 面板，让 graph 成为主体。
- graph 区域放大，减少“图很小、面板很重”的割裂感。
- 综合练习入口放入 graph 左下角，作为图内特殊入口，而不是独立详情面板按钮。

v0.0.21 校准：

- 常驻练习区先展示练习模式，再展示当前通路下的“开始练习 / 查看弱点”动作。
- “图选”归入练习模式选择层，作为另一种选择模式的入口。
- “开始练习 / 查看弱点”归入当前通路详情层，避免动作按钮抢在模式选择之前出现。

v0.0.22 校准：

- 移除常驻区内“当前通路：...”说明块，避免与练习模式高亮状态重复。
- 练习模式说明改为按钮 hover tooltip，按需查看。
- 常驻区只保留模式选择和当前模式下的“开始练习 / 查看弱点”动作。

v0.0.23 校准：

- “图选”不再作为练习模式按钮队列中的独立项。
- “图选”移动到“练习模式”标题旁，作为选择器的辅助入口。
- 模式按钮行只保留真实练习模式，减少一行控件的视觉重量。

v0.0.24 校准：

- “练习 / 查看弱点”改为当前通路下的 tab，而不是并列动作按钮。
- 练习模式选择器始终位于上方，tab 内容统一显示在其下方。
- 练习 tab 和弱点 tab 都采用“左侧主体 + 右侧详情”的下方内容布局，避免右侧详情位置在两个 tab 间跳动。

v0.0.25 校准：

- 顶部 Header 压缩为紧凑工具栏，移除日常练习中重复阅读价值较低的说明文案。
- 全局导航、调性和唱名显示控件高度收紧，减少首屏纵向占用。
- 练习模式区从大卡片降级为紧凑工具栏，降低 padding 和视觉重量。
- 将“练习 / 查看弱点”tab 与练习模式标题同区展示，让题目区更早进入首屏。

待发布校准：

- 日常练习模式改为“训练组 + 方向”两层选择，避免 8 个以上具体题型平铺。
- 新增 `唱名 -> 音名` 与 `唱名 -> 位置` 两条反向通路。
- 将“位置音名 / 位置唱名”合并为一级“指板记忆”，二级横向分为音名组与唱名组。
- 唱名组内提供当前调选择，再提供 `位置 -> 唱名`、`唱名 -> 位置`、`混合`。
- 其它训练组内提供混合方向：音名唱名混合、六线谱混合。
- 图选弹窗继续呈现完整关系图，日常工具栏只承担快速切换。

暂不完成：

- 新增五度圈、音阶表等速查工具。
- 为所有通路补齐弱点统计。
- 制作复杂动效或力导向图。
- 改造底层题库为完全图模型。

## 9. 验证标准

用户刷新页面后，应能快速回答：

- 我现在是在练习，还是在速查？
- 当前是什么调？
- 当前唱名显示方式是什么？
- 我选择的是哪条认知通路？
- 我现在是要开始练习，还是查看这条通路的弱点？

E2E 重点覆盖：

- 默认进入练习页。
- 通路图显示四个节点和当前支持的通路。
- 点击通路可以切换当前练习类型。
- 点击“开始练习”进入对应题型并重开本轮。
- 点击“查看弱点”进入对应弱点视图。
- 指板速查出现在速查页。
- 切换调性后保留当前通路。
- 唱名显示模式继续影响练习、反馈、速查与弱点文本。

---

## 10. 后续演进

当更多练习通路拥有稳定数据后，可以考虑：

- 在通路图边上显示弱点热度或最近练习状态。
- 在通路操作区提供“针对弱点复练”入口。
- 将综合练习配置为“选择参与混合的通路”。
- 新增速查工具，例如五度圈、调内音阶、音程关系图。
- 当数据规模足够时，再抽象全局学习报告，而不是提前建立庞大的数据中心。
