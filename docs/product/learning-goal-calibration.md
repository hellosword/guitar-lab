# Guitar Lab — 个人学习目标校准

> 日期：2026-05-01  
> 目的：从真实学习需求反推 MVP 范围，避免产品设计过度发散。

---

## 1. 当前学习阶段

用户当前处于吉他初学者阶段：

- 能够弹奏简单歌曲
- 主要依赖手部肌肉记忆完成演奏
- 能看六线谱，并能把六线谱数字对齐到琴弦和品格位置
- 已有一定 C 调熟悉度，但 G 调不够熟练

当前问题不是“完全不会弹”，而是：

> 能把谱子弹出来，但还不能快速知道自己弹的是什么音，以及这个音在当前调里唱什么。

---

## 2. 核心痛点

### 2.1 六线谱到指板位置的反应速度不够快

看到六线谱时，用户可以理解其对应的琴弦和品格，但反应速度仍不够快。

这说明第一层能力已经存在，但尚未自动化：

```text
P_tab -> P_board
六线谱位置 -> 实际指板位置
```

### 2.2 位置到音名/唱名无法即时反应

用户弹奏时常常知道“手应该按哪里”，但不能立即说出：

- 这个位置是什么音名
- 这个音在当前调里唱 Do/Re/Mi/Fa/Sol/La/Si 的哪一个

当前反应链路大致是：

```text
看到六线谱 -> 找到琴弦和品格 -> 弹奏
                         -> 几秒后才想起音名/唱名
```

目标是把它训练成近似自动化的链路：

```text
看到六线谱/指板位置 -> 立即知道音名 -> 立即知道当前调唱名 -> 能唱出旋律
```

### 2.3 G 调尤其需要优先训练

用户明确提到：

- C 调相对更熟
- G 调不如 C 调熟练

因此 MVP 不能只从 C 调开始，也应尽早纳入 G 调，尤其是 G 大调中的 F#。

---

## 3. 第一阶段学习目标

第一阶段不追求完整覆盖 Level 1~6，而是优先解决以下能力：

### 目标 A：六线谱位置反应自动化

看到简单六线谱位置时，能够快速映射到实际指板位置。

对应训练：

```text
P_tab -> P_board
```

### 目标 B：位置到音名自动化

看到或按到某个位置时，能够立即说出音名。

对应训练：

```text
P_board -> N
P_tab -> N
```

### 目标 C：位置到 G 调唱名自动化

在 G 大调语境下，看到或按到某个位置时，能够立即知道唱名。

对应训练：

```text
P_board + K -> S
P_tab + K -> S
```

其中 `K` 第一阶段重点为：

```text
C 大调、G 大调
```

但 G 大调应被视为重点训练对象，而不是 C 大调之后很久才解锁的内容。

### 目标 D：帮助旋律内化，而不是只记动作

训练的最终目的不是答题得分本身，而是让用户能更流畅地唱出自己弹奏的旋律。

因此题目反馈应尽量帮助用户建立三者绑定：

```text
位置 -> 音名 -> 唱名
```

---

## 4. MVP 范围建议

第一版 MVP 应服务“六线谱/位置到音名与唱名的快速反应”，暂不做音程、和弦、音阶把位等高阶内容。

### 4.1 推荐训练范围

```text
调性：C 大调、G 大调
范围：0-5 品
媒介：指板图 + 简单六线谱
```

`0-5 品` 的原因：

- 覆盖初学者常见开放把位
- 包含 G 大调关键音 F#
- 范围足够小，适合建立第一批自动化反应

### 4.2 推荐首批题型

| 优先级 | 题型 | 目的 |
|--------|------|------|
| P0 | `P_board -> N` | 看到指板位置，立即回答音名 |
| P0 | `P_board + K -> S` | 看到指板位置，立即回答当前调唱名 |
| P1 | `P_tab -> N` | 从六线谱数字直接反应音名 |
| P1 | `P_tab + K -> S` | 从六线谱数字直接反应唱名 |
| P2 | `N -> P_board` | 反向定位音名，补强指板地图 |
| P2 | `S + K -> P_board` | 反向定位唱名，补强调内听感 |

### 4.3 暂缓内容

以下内容虽然重要，但不进入第一版 MVP：

- 音程几何
- 和弦内音定位
- CAGED / 把位音阶自由
- 多声部指弹分析
- 节拍器与节奏打分
- 复杂关卡地图
- 排行榜、成就、社交分享

这些能力应在用户已经建立基础 `位置 -> 音名 -> 唱名` 自动化后再加入。

---

## 5. MVP 成功标准

第一版不以功能数量作为成功标准，而以真实练习效果作为标准。

建议用 7 天验证：

1. 用户是否愿意连续打开练习
2. G 大调 0-5 品的音名/唱名反应是否明显变快
3. 看简单六线谱时，是否能更快说出自己弹的音
4. 弹熟悉旋律时，是否能更流畅地唱出唱名

可量化指标：

| 指标 | 初始状态 | 目标 |
|------|----------|------|
| G 大调位置到音名平均反应时间 | 待测 | 降低 30% |
| G 大调位置到唱名平均反应时间 | 待测 | 降低 30% |
| 0-5 品答题正确率 | 待测 | 达到 85% 以上 |
| F# 相关题目正确率 | 待测 | 达到 80% 以上 |

---

## 6. 对产品设计的影响

基于本次校准，Guitar Lab 的第一阶段应从“完整训练系统”收缩为：

> 一个帮助初学者把六线谱/指板位置快速转化为音名与唱名的练习器。

因此近期优先级应调整为：

1. 指板图 SVG
2. 简单六线谱展示
3. 音名选择器
4. 唱名选择器
5. C/G 大调 0-5 品题目生成
6. 即时反馈与错题重复
7. 简单记录反应时间与正确率

这比完整 Level 1~6、复杂题库扩展和移动端打包更重要。
