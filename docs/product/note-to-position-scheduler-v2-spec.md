# 音名定位调度器 V2 规格

> 版本目标：v0.0.12
> 当前状态：已实现于 v0.0.12
> 最后更新：2026-05-03

---

## 1. 背景

“音名定位”题型的核心任务是：

> 给出一个音名，让用户在空指板上找出当前调性、当前品位范围内的所有对应位置。

v0.0.10 和 v0.0.11 已经让系统能够根据弱位置反推出题面音名，并避免同一个音名连续刷屏。但当前调度规则仍然偏硬：

- 弱点强化比例通过固定节奏控制。
- 普通覆盖和弱点强化是两套分支。
- 音名冷却是额外补丁，而不是自然来自覆盖权重。

本规格定义一套更薄的规则层：**先按位置权重决定下一题为什么出现，再按音名组织这题怎么考。**

---

## 2. 目标

### 2.1 产品目标

- 弱位置更容易被抽到，但不会机械重复同一个音名。
- 普通覆盖自然发生，而不是依赖“每几题插一题普通题”。
- 熟练位置不会永久免考，也不会每次都消耗注意力。
- 未知位置会自然进入测量，让系统逐步补齐用户能力画像。
- 规则足够薄，后续可以推广到唱名、六线谱或其它题型。

### 2.2 学习目标

优先服务当前核心学习需求：

> 建立开放把位内“音名 -> 指板位置”的快速反应，并进一步支撑“位置 -> 音名 -> 首调唱名”的整体直觉。

v0.0.12 只作用于“音名定位”模式，不改变其它题型。

---

## 3. 非目标

v0.0.12 不做：

- 新增关卡页或规则小练习。
- 改造全部题型的调度系统。
- 引入机器学习模型。
- 使用真实时间判断遗忘压力。
- 新增复杂后台配置页面。
- 改变当前音名定位的主要 UI 形态。

---

## 4. 核心模型

每个候选位置都有一个综合权重：

```text
综合权重 = 静态权重 + 动态权重
```

### 4.1 静态权重

静态权重来自长期练习记忆，表达“这个位置本身是不是薄弱”。

当前可使用的数据包括：

- `weaknessScore`
- `wrongCount`
- `slowCount`
- `fastCorrectStreak`
- `attempts`

第一版静态权重可以沿用现有弱点加权方式：

```text
staticWeight = baseWeight + min(weaknessScore, maxWeaknessBonus)
```

### 4.2 动态权重

动态权重只服务于“本轮覆盖节奏”。

它不表达长期熟练度，也不表达遗忘压力，只表达：

> 这个位置在本轮里有多需要被覆盖。

规则：

- 一轮开始时，所有候选位置的动态权重为 0。
- 每生成一道音名定位题，未被本题音名覆盖的位置动态权重增加。
- 本题音名下所有位置的动态权重清空。
- 动态权重有上限，避免压过长期弱点数据。

这样可以避免把“很久没考过”重复塞进熟练复查概率里。

---

## 5. 出题流程

每次生成音名定位题时：

1. 收集当前调性、当前品位范围内的所有候选位置。
2. 为每个候选位置计算 `静态权重 + 动态权重`。
3. 按综合权重抽中一个位置。
4. 用这个位置反推出题面音名。
5. 将同音名下所有位置分成三类：
   - 弱位置
   - 未知位置
   - 熟练位置
6. 决定本题哪些位置需要用户点击，哪些位置直接提示。
7. 生成题目后，更新本轮动态权重。

示例：

```text
抽中位置：4 弦 2 品
该位置音名：E
生成题目：在 0-4 品内找出所有 E
```

---

## 6. 同音名位置的答题规则

题面仍然保持整体感：

> 找出当前范围内所有这个音名的位置。

但实际点击目标按掌握状态决定。

### 6.1 弱位置

弱位置必须点击。

判断依据：

```text
weaknessScore > 0
```

### 6.2 未知位置

未知位置必须点击。

未知位置指没有足够练习记录的位置。第一版可简单定义为：

```text
没有对应 mastery entry
```

这样系统会继续补齐用户能力画像。

### 6.3 熟练位置

熟练位置不再永远提示，而是按复查概率决定：

- 抽中复查：需要点击。
- 未抽中复查：直接显示音名圆点提示。

这样熟练位置会低频复查，避免退化；同时不会每次都打扰用户。

第一版可沿用当前“已掌握”的最低标准：

```text
weaknessScore <= 0 且 fastCorrectStreak >= 2
```

---

## 7. 熟练位置复查概率

熟练位置复查概率用于回答：

> 这个已经熟练的位置，这次还要不要顺手考一下？

复查概率在配置范围内变化：

```text
retestRate = minRate ~ maxRate
```

影响因素：

- `attempts` 少：复查概率更高。
- `fastCorrectStreak` 多：复查概率更低。
- `wrongCount` 高：复查概率更高。
- `slowCount` 高：复查概率更高。
- `weaknessScore` 高：复查概率更高。

不纳入的因素：

- 距离上次真实时间。
- 距离上次经过多少题。

原因：覆盖间隔已经由动态权重处理，复查概率只负责评估掌握质量。

## 8. 建议配置

```ts
noteToPositionScheduler: {
  staticWeight: {
    baseWeight: 1,
    maxWeaknessBonus: 3,
    maxFinalWeight: 4,
  },
  dynamicWeight: {
    gainPerQuestion: 1,
    maxBonus: 4,
    resetScope: 'same-note',
  },
  masteredRetest: {
    minRate: 0.15,
    maxRate: 0.75,
    fastStreakTarget: 5,
    lowConfidenceAttemptCount: 3,
  },
}
```

参数含义：

- `baseWeight`：每个候选位置的基础出现权重。
- `maxWeaknessBonus`：长期弱点最多贡献多少额外权重。
- `maxFinalWeight`：静态权重上限。
- `gainPerQuestion`：每生成一道音名定位题，未覆盖位置增加多少动态权重。
- `maxBonus`：动态权重上限。
- `resetScope`：生成某音名题后，清空同音名位置的动态权重。
- `minRate`：熟练位置最低复查率，避免永久免考。
- `maxRate`：熟练位置最高复查率，避免“熟练”位置仍然过度打扰。
- `fastStreakTarget`：连续快速答对达到该值时，认为近期稳定性接近满值。
- `lowConfidenceAttemptCount`：样本少于该值时，即使近期答对，也提高复查概率。

## 9. 确定性随机

调度器需要使用确定性随机，而不是每次刷新都变化的 `Math.random()`。

原因：

- 同一份练习数据生成的题目应尽量稳定。
- E2E 测试需要可预测。
- 用户刷新页面时，不应感觉系统规则飘忽。

随机种子可以来自：

```text
题目 index + 当前调性 + 候选位置 id + 当前模式
```

熟练位置复查也应使用确定性随机：

```text
题目 id + 位置 id
```

---

## 10. 预期体验

实施后，用户体感应为：

- 弱点更容易回来，但不会连续机械刷同一个音名。
- 如果 E 已经出过一题，E 下所有位置的动态覆盖压力会被清空，下一题自然更倾向其它音名。
- 熟练位置有时会被复查，有时会直接提示。
- 熟练位置一旦答慢或答错，后续复查概率会自然回升。
- 新位置不会被永久忽略，会被纳入点击目标并产生数据。

---

## 11. 验收标准

- 音名定位不再使用 `weakFocusCycleSize / broadCoverageRemainder / noteCooldownCount` 这类硬节奏规则。
- 第一题能从弱位置或普通覆盖权重中稳定生成。
- 同一个音名被生成后，该音名下所有位置的动态权重清空。
- 弱位置和未知位置必须点击。
- 熟练位置按复查概率决定点击或提示。
- 调度逻辑有单元测试或 E2E 覆盖，至少验证：
  - 多个弱 E 位置不会导致 E 连续刷屏。
  - 熟练位置会出现“提示”和“复查”两种结果。
  - 未知位置默认纳入点击目标。
  - 当前已有音名定位交互不退化。
